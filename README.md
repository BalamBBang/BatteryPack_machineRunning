# 🔋 배터리팩 이상감지 시스템

KAMP 전자부품(배터리팩) 품질보증 AI 데이터셋을 활용한 **비지도 이상탐지** 프로젝트입니다.
정상 배터리 데이터만으로 학습하여 용량불량, 용접불량, 센싱와이어불량, 센서불량 등의 고장을 탐지합니다.

---

## 📂 데이터셋

| 항목 | 내용 |
|------|------|
| 출처 | [KAMP 전자부품(배터리팩) 품질보증 AI 데이터셋](https://www.kamp-ai.kr/aidataDetail?AI_SEARCH=&page=2&DATASET_SEQ=58) |
| github | [이스트캠프 AI모델개발 15기 배터리팩 품질보증 발표조](https://github.com/BalamBBang/BatteryPack_machineRunning.git) |
| 구성 | 충전(chg) / 방전(dchg) CSV 파일 |
| Train | 정상 데이터만 포함 (배터리 ID별 chg/dchg 쌍) |
| Test | OK(정상) 4개 + NG(이상) 5개, 총 9개 파일 |
| 컬럼 수 | 231개 (셀 전압, 온도, 전류, 전력, 시간 등) |

### 고장 유형

- **용량불량**: 특정 셀에서 충전 중 전압 급변
- **용접불량**: 특정 셀의 전압 미측정 또는 전체 전압 저하
- **센싱와이어불량**: 인접 셀 전압 역전
- **센서불량**: 센서 데이터 자체의 이상

## 🏗️ 파이프라인 구조

```
원본 CSV
  │
  ▼
[1] 전처리
  │  단일값/결측치/이상치 제거, 불필요 컬럼 삭제 (Date 등)
  │  사용 컬럼: Time + 셀 전압(CV) + 온도(T) + 전류/전력/전압
  │
  ▼
[2] 파생변수 생성 (행 단위 14개 특징)
  │
  ▼
[3] 슬라이딩 윈도우
  │  window=30, step=1, active 구간만 사용
  │
  ▼
[4] StandardScaler → PCA(64차원) → One-Class SVM
  │
  ▼
[5] 후처리
  │  ├─ 기본 후처리: 잡음 제거, 구간 병합, 약한 구간 제거
  │  └─ 고장 인지 후처리: 고장 단서 점수 반영한 구간 보강/기각
  │
  ▼
최종 이상 판정 + 고장 유형 추정
```

## 🤖 모델

### One-Class SVM (최종 채택)

비지도 이상탐지 모델로, **정상 데이터만으로 학습**하여 정상 영역의 경계를 학습합니다.

- **커널**: RBF
- **gamma**: `scale` (자동)
- **학습 데이터**: 정상 배터리 38개 (충전/방전 각각)
- **차원 축소**: PCA 64차원 (분산 보존 충전 98.9%, 방전 98.3%)

### 이전 시도

| 모델 | 결과 |
|------|------|
| LSTM Autoencoder | 추세만 반영, 크기 정보 미반영 → 불충분 |
| Hybrid (IsolationForest + PCA) | 중간 단계 → One-Class SVM으로 대체 |

## 🔍 하이퍼파라미터 탐색 (Grid Search)

4단계 순차 탐색을 수행합니다. (`auto_calibrate_mode` 함수)

### 탐색 순서 및 그리드

| 단계 | 대상 | 탐색 범위 | 후보 수 |
|------|------|-----------|---------|
| 1 | PCA 차원 | 4, 6, 8, 10, 12, 16, 20, 24, 28, 32, 40, 48, 56, 64 | 14 |
| 2 | OCSVM nu + 임계값 분위수 | nu: 0.005~0.15 (10종) × threshold_q: 0.98~0.9995 (9종) | 90 |
| 3 | 기본 후처리 6개 파라미터 | low_ratio, merge_gap, min_len, persistent_ratio, head_ratio, tail_ratio | 최대 4,800 |
| 4 | 고장 인지 후처리 | reject_support, reinforce_multiplier 등 | 381 |

### 선택 기준

```
objective = synthetic_F1 - normal_alarm_rate
```

검증용 데이터(calibration set, 20%)와 합성 이상 데이터로 평가합니다.

### 최종 선택된 설정

| 모드 | PCA 차원 | nu | 임계값 분위수 | 정상 경보율 |
|------|----------|-----|-------------|------------|
| 충전 | 64 | 0.1000 | 0.9800 | 0.000 |
| 방전 | 64 | 0.0050 | 0.9900 | 0.000 |

## 🔧 후처리

### 기본 후처리 (`generalized_postfilter`)

| 파라미터 | 충전 | 방전 | 설명 |
|----------|------|------|------|
| `low_ratio` | 0.55 | 0.45 | 하한 임계값 비율 |
| `merge_gap_rows` | 30 | 30 | 가까운 구간 병합 거리 |
| `min_interval_len` | 60 | 10 | 최소 이상 구간 길이 |

처리 흐름: 짧은 잡음 제거 → 가까운 구간 병합 → 약한 구간 제거

### 고장 인지 후처리 (`fault_aware_postfilter`)

고장 단서 점수(fault score)를 반영하여:
- 고장 가능성이 높으면 구간을 보강
- 고장 가능성이 낮으면 구간을 기각
